## Root Cause Analysis

**Gap 1 root cause**: ClientBuilder.build() calls `new OpenClawClient(clientConfig)` without the `_internal` argument. The OpenClawClient constructor detects `_internal` is falsy and takes the backward-compatible path which re-creates all managers inline. The extraction was incomplete — manager creation was NOT moved to ClientBuilder.

**Gap 2 root cause**: createClient was kept for backward compatibility (per STATE.md decision record line 51). D-03 specified removal but it was not honored.

## What Needs to Change

### Gap 1 Fix: Complete Manager Extraction

1. ClientBuilder.build() must create all managers (transport, authHandler, reconnectMgr, connectionManager, requestManager, eventManager, protocolNegotiator, policyManager, stateMachine)
2. ClientBuilder.build() must pass these managers via the `_internal` argument to OpenClawClient constructor
3. OpenClawClient constructor's `_internal` path must be completed (currently sets managers but NOT API namespaces — those are created in the constructor body regardless of _internal)

### Gap 2 Fix: Remove createClient

1. Remove `export { ClientBuilder, createClient }` from client-builder.ts
2. Remove `export { ClientBuilder, createClient }` from index.ts (line 11)
3. Keep only `export { ClientBuilder }`
